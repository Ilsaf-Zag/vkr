# mobile-driver-app

Мобильное приложение водителя для внутренней информационной системы ООО «АЗЫК».

Технология: Flutter.

## Назначение

Приложение реализует пошаговый workflow смены водителя:

1. авторизация;
2. проверка активного план-наряда;
3. открытие путевого листа;
4. фото и подтверждение начального одометра;
5. предрейсовый медосмотр;
6. предрейсовый техосмотр;
7. имитация первой печати;
8. начало смены;
9. заправки и GPS;
10. завершение рейса;
11. фото и подтверждение конечного одометра;
12. послерейсовые проверки;
13. итоговая печать;
14. закрытие смены и очистка локальной сессии.

## Запуск на устройстве или эмуляторе

```bash
flutter pub get
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000/api
```

Для Android-эмулятора используется адрес `http://10.0.2.2:8000/api`, потому что `localhost` внутри эмулятора указывает на сам эмулятор.

Для iOS Simulator или desktop:

```bash
flutter run --dart-define=API_BASE_URL=http://localhost:8000/api
```

## Сборка APK

Для реального телефона backend должен быть доступен телефону по IP компьютера в локальной сети. Соберите APK с адресом API:

```bash
flutter pub get
flutter build apk --debug --dart-define=API_BASE_URL=http://192.168.1.25:8000/api
```

APK появится здесь:

```text
mobile-driver-app/build/app/outputs/flutter-apk/app-debug.apk
```

Вместо `192.168.1.25` нужно подставить IP компьютера, на котором запущен Docker.

## Проверка OCR-сценария

На этапах начала и завершения смены приложение попросит сфотографировать одометр. Можно использовать камеру или выбрать фото из галереи. После распознавания водитель подтверждает значение или исправляет его вручную.

Тестовый водитель:

- логин: `driver1`;
  - пароль: `driver123`.
