# mobile-driver-app

Мобильное приложение водителя для внутренней информационной системы ООО «АЗЫК».

Технология: Flutter.

## Назначение

Приложение реализует пошаговый workflow смены водителя:

1. авторизация;
2. проверка активного план-наряда;
3. открытие путевого листа;
4. предрейсовый медосмотр;
5. предрейсовый техосмотр;
6. имитация первой печати;
7. начало смены;
8. заправки и GPS;
9. завершение рейса;
10. послерейсовые проверки;
11. итоговая печать;
12. закрытие смены и очистка локальной сессии.

## Запуск на устройстве или эмуляторе

```bash
flutter pub get
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000/api
```

Для Android-эмулятора используется адрес `http://10.0.2.2:8000/api`, потому что `localhost` внутри эмулятора указывает на сам эмулятор.

Для iOS Simulator или desktop:

```bash
flutter run --dart-define=API_BASE_URL=http://localhost:8000/api
```

## Сборка APK через Docker Compose

```bash
docker compose --profile mobile run --rm mobile-build
```

Flutter локально ставить не нужно. Контейнер сам создаст Android-каркас, если его еще нет, скачает зависимости и соберет APK.

APK появится в `mobile-driver-app/build/app/outputs/flutter-apk/app-debug.apk`.

Тестовый водитель:

- логин: `driver1`;
  - пароль: `driver123`.
